# Ассистент Cleaner (расширение Chrome)

Расширение для работы с [Cleaner](https://clr.env7.biz) и Pyrus: группировка OpenEnds, проверка открытых вопросов с ручной чисткой и перенос данных проекта из Pyrus в форму Cleaner.

---

## Где работает

| Сайт / страница | Режим |
|-----------------|--------|
| `clr.env7.biz/lk/Project/Edit/<id>#openends` | Группировка OpenEnds |
| `clr.env7.biz/lk/openends2/verifymain` | Проверка OPENEND-OB, ручная чистка |
| `clr.env7.biz/lk`, `clr.env7.biz/lk/projects` | Автозаполнение формы по ссылке из Pyrus |
| `pyrus.com/t#id...` (карточка задачи) | Кнопка «В CleanerUI» |

---

## Функционал

### 1. Группировка OpenEnds (вкладка #openends проекта)

- **Панель расширения** (иконка в тулбаре Chrome на странице редактирования проекта с хешем `#openends`):
  - Поиск дублей по точному совпадению или по похожести текста (порог 0.50–1.00).
  - Режим «Разные переменные отдельно» — группы строятся с учётом переменной (Q1_1_other, Q1_2_other и т.д.).
  - Кнопки **«Сгруппировать все»** и **«Снять подсветку»**.
  - Сохранение групп в локальное хранилище; синхронизация при ручной группировке/разгруппировке через кнопки интерфейса Cleaner (group/ungroup).

**Как пользоваться:** откройте `https://clr.env7.biz/lk/Project/Edit/<id>#openEnds`, нажмите иконку расширения, при необходимости выберите режим и порог, затем «Сгруппировать все».

### 2. Проверка OPENEND-OB (страница verifymain)

- Загрузка выгрузки OpenEnds (XLSX) по проекту для привязки ответов к респондентам.
- Колонка **«Другие ответы»** с кнопкой **«Посмотреть»** — открывается модальное окно с ответами респондента и чекбоксом **«В ручную чистку»** (при наличии токена ручной чистки и если респондент ещё не в ручной чистке/не помечен как некорректный).
- Кнопка **«Проверить страницу»**:
  - выполняет стандартную проверку страницы;
  - отправляет выбранных в модалке респондентов в ручную чистку (в буфер и при наличии — через API).
- Подсветка строк по кодам причин (некорректный, спидстер, ручная чистка и т.д.).

**Ручная чистка:** чтобы чекбокс «В ручную чистку» был активен, нужно один раз открыть вкладку **«Ручная чистка»** этого проекта, нажать **«Добавить брак»** и сохранить (расширение запомнит токен).

### 3. Перенос из Pyrus в Cleaner

- На странице задачи Pyrus рядом с кнопкой «Новый клиент» / «Чистилка» появляется кнопка **«Сетап Чистилки»**.
- По нажатию открывается `https://clr.env7.biz/lk?qga_autofill=1` в новой вкладке; форма на странице Cleaner заполняется данными из карточки Pyrus:
  - **Наименование проекта**
  - **ID проекта** (номер в панели)
  - **План**
  - **База ОД**

Метки полей в Pyrus распознаются по синонимам (например: «название проекта», «project name», «база од», «sawtooth» и т.д.).

### 4. Автозаполнение формы на clr.env7.biz/lk

- При переходе на `clr.env7.biz/lk` или `clr.env7.biz/lk/projects` с параметром `qga_autofill=1` (так открывается ссылка после нажатия «В CleanerUI» в Pyrus) расширение подставляет сохранённые данные проекта в форму.

---

## Файлы

- `chrome-extension/manifest.json` — манифест расширения (Manifest V3).
- `chrome-extension/content.js` — основной скрипт (OpenEnds, проверка, Pyrus, форма Cleaner).
- `chrome-extension/background.js` — сервис-воркер.
- `chrome-extension/xlsx.full.min.js` — разбор выгрузки OpenEnds (XLSX).
- `chrome-extension/icons/` — иконки расширения и статусов (некорректный, спидстер, ручная чистка и т.д.).

---

## Установка

1. Откройте `chrome://extensions/`.
2. Включите **Режим разработчика**.
3. Нажмите **Загрузить распакованное расширение**.
4. Укажите папку **`chrome-extension`** этого репозитория.
