# Privacy Policy

Last updated: 2026-04-16

`Cleaner Assistant` is an internal browser extension used to support workflows in `Cleaner` and `Pyrus`.

## What data the extension accesses

The extension accesses only the information required to perform its user-facing features on the supported domains:

- page content on `https://clr.env7.biz/*`;
- page content on `https://pyrus.com/*` and `https://www.pyrus.com/*`;
- locally stored extension settings and temporary workflow state;
- files explicitly selected by the user for processing inside the extension, such as OpenEnds XLSX exports.

## What data the extension stores

The extension stores data locally in the browser only when required for its functionality. This may include:

- UI settings;
- cached intermediate project data;
- grouped OpenEnds state;
- selected IDs for verification-related workflows;
- favorite projects and related local preferences.

This local data remains in the user's browser profile and is not sent to developer-owned servers.

## What data the extension transmits

The extension sends requests only to the target web applications the user is already working with:

- `https://clr.env7.biz/*`
- `https://pyrus.com/*`
- `https://www.pyrus.com/*`

These requests are performed only to support user-initiated product features inside those systems, such as reading current page state, loading related data, or submitting actions in Cleaner.

The extension does not send collected browsing data, personal data, or project data to any third-party analytics, advertising, or developer-operated backend.

## Tokens and authenticated requests

For some Cleaner workflows, the extension may use page-level anti-forgery or session-bound request tokens that already exist in the user's authenticated session. These tokens are used only to complete actions requested by the user inside the target application and are not transferred to unrelated services.

## Data sharing

The extension does not sell user data.

The extension does not transfer user data to third parties, except for requests made directly to the supported target services as part of the extension's core functionality.

## Data retention

Any locally stored workflow data remains in the browser until it is overwritten, cleared by the extension logic, or removed by the user through browser storage cleanup or extension removal.
